# 项目详解

> 这是一份给"懂前端但没用过 Hexo / Vercel"的工程师看的项目说明。读完你就能完全掌握这个博客的运转机制。

---

## 一、一句话总结

这是一个**用 Hexo（静态站点生成器）写的个人博客**，主题是 **Butterfly**，托管在 **Vercel** 上。日常用法：

> 写一个 Markdown 文件 → `git push` → 30 秒后自动上线。

---

## 二、用前端术语类比

| 这个项目里的东西 | 你熟悉的前端等价物 |
|---|---|
| Hexo | 类似 Next.js 的 SSG 模式 / Vite SSG / Astro，把数据 + 模板编译成纯静态 HTML |
| Butterfly 主题 | 一套写好的 Pug 模板 + Stylus 样式 + 前端 JS（相当于一整套 UI 组件库 + 页面布局）|
| `_posts/*.md` | "数据源"——相当于 CMS 里的内容，构建时被读进来渲染成页面 |
| `_config.yml` | 项目全局配置，像 `next.config.js` 但是 YAML 写的 |
| `public/` | 构建产物，相当于 `dist/` 或 `.next/out/` |
| Vercel | 跟你认识的 Vercel 一样：连 GitHub → 监听 push → 跑 build → CDN 部署 |
| `vercel.json` | 部署时告诉 Vercel "用什么命令构建、产物在哪" |

**关键认知**：这个项目里**没有运行时**——访客访问的就是一堆静态 HTML/CSS/JS，没有 Node 服务器。所有动态内容都是构建时生成的。

## 三、整体数据流

```
┌─────────────────────┐
│  source/_posts/*.md │  你写的内容
│  source/about/*.md  │
│  source/img/*       │
└──────────┬──────────┘
           │
           │  hexo generate（构建）
           │   - 读 _config.yml 拿配置
           │   - 读 _config.butterfly.yml 拿主题配置
           │   - 用 Butterfly 主题的 Pug 模板渲染
           │   - 跑各种 generator 插件（首页、归档、标签、RSS、sitemap、search.xml）
           │
           ▼
┌─────────────────────┐
│  public/            │  纯静态文件
│   ├── index.html    │
│   ├── posts/...     │
│   ├── archives/...  │
│   ├── css/, js/     │
│   ├── search.xml    │
│   ├── sitemap.xml   │
│   └── atom.xml      │
└──────────┬──────────┘
           │
           │  git push origin main
           │
           ▼
┌─────────────────────┐
│  GitHub             │
└──────────┬──────────┘
           │
           │  webhook 触发 Vercel
           │
           ▼
┌─────────────────────┐
│  Vercel             │  在云端跑：
│   1. npm install    │   - 装依赖
│   2. npm run build  │   - 跑 hexo generate
│   3. 上传 public/   │   - 推到全球 CDN
└──────────┬──────────┘
           │
           ▼
   读者看到新文章
```

**重点**：本地的 `public/` **不进 git**（见 `.gitignore`）。Vercel 每次都从源文件重新构建。

## 五、关键文件深度解读

### 5.1 `package.json`

```json
{
  "scripts": {
    "build": "hexo generate",     // 生成静态站到 public/
    "clean": "hexo clean",        // 清掉 public/ 和 db.json
    "server": "hexo server",      // 本地预览（http://localhost:4000）
    "new": "hexo new"             // 新建文章
  },
  "dependencies": {
    "hexo": "^7.3.0",                       // 静态站生成器本体
    "hexo-theme-butterfly": "^4.13.0",      // 主题
    "hexo-generator-archive": "...",        // 归档页生成器
    "hexo-generator-category": "...",       // 分类页
    "hexo-generator-tag": "...",            // 标签页
    "hexo-generator-feed": "...",           // RSS / Atom 输出
    "hexo-generator-index": "...",          // 首页 + 分页
    "hexo-generator-search": "...",         // 搜索数据 search.xml（前端搜索依赖）
    "hexo-generator-sitemap": "...",        // sitemap.xml（SEO）
    "hexo-renderer-ejs": "...",             // 模板引擎
    "hexo-renderer-marked": "...",          // Markdown 渲染
    "hexo-renderer-pug": "...",             // Butterfly 主题用 Pug 模板
    "hexo-renderer-stylus": "...",          // Butterfly 主题用 Stylus 样式
    "hexo-server": "...",                   // 本地 dev server
    "hexo-wordcount": "..."                 // 字数统计
  }
}
```

**Hexo 的插件机制**：把功能拆成一堆 npm 包。删一个 `hexo-generator-search` 搜索就废了——这就是上次搜索失效的原因。

### 5.2 `_config.yml`（Hexo 主配置）

按区块的作用：

| 区块 | 干啥 |
|---|---|
| `title / subtitle / description / keywords / author` | 站点元信息，写 HTML `<meta>` 用 |
| `url` | 站点根 URL，影响 RSS / sitemap 里的绝对链接 |
| `permalink: posts/:title/` | 文章 URL 规则。当前文章访问路径形如 `/posts/hello-world/` |
| `pretty_urls` | URL 美化（去掉 `.html` 和 `/index`）|
| `source_dir / public_dir` | 输入目录、输出目录 |
| `post_asset_folder: true` | 允许给文章建同名文件夹放图片 |
| `syntax_highlighter: highlight.js` | 代码高亮引擎 |
| `index_generator.per_page: 10` | 首页每页 10 篇 |
| `search` | 让 `hexo-generator-search` 输出 `search.xml`（前端搜索数据源）|
| `sitemap` | SEO 站点地图 |
| `feed` | RSS / Atom 订阅 |
| `theme: butterfly` | 启用的主题名（对应 `node_modules/hexo-theme-butterfly`）|

### 5.3 `_config.butterfly.yml`（主题配置，最常调）

这是最大也最重要的一个文件（800+ 行）。所有视觉、特性开关都在这。常调的几个区块：

```yaml
nav:                  # 导航栏
menu:                 # 顶部菜单项
social:               # GitHub / Email / RSS 等社交图标
avatar:               # 头像 + 是否旋转
index_img:            # 首页顶部 banner 图
default_top_img:      # 文章默认顶部图
cover:                # 列表页封面图配置
post_meta:            # 文章上下方的 meta 信息（作者、日期、字数等）
toc:                  # 目录设置
post_copyright:       # 文末版权信息
related_post:         # 相关文章
footer:               # 页脚
aside:                # ★ 侧边栏（卡片、公告、最新文章等都在这）
darkmode:             # 暗色模式
katex / mathjax:      # 数学公式（已开 KaTeX）
mermaid:              # 流程图（已开）
local_search:         # 本地搜索（已开）
comments:             # 评论系统（未启用，可选 Twikoo / Waline / Giscus 等）
google_analytics:     # 分析工具
theme_color:          # 主题色（被 custom.css 覆盖）
font:                 # 字体
subtitle:             # ★ 首页打字机效果文字
wordcount:            # 字数统计（已开）
fancybox:             # 图片点开放大（已开）
instantpage: true     # 鼠标悬停时预加载下一页（性能优化）
inject:               # ★ 自定义 CSS / JS 注入点
```

**关键点 `inject`**：

```yaml
inject:
  head:
    - <link rel="preconnect" href="https://fonts.googleapis.com">
    - <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
    - <link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap">
    - <link rel="stylesheet" href="/css/custom.css">
```

这是**当前项目自定义样式的入口**——通过 inject 把 Inter 字体和 `custom.css` 加载进去。

### 5.4 `vercel.json`

```json
{
  "buildCommand": "npm run build",   // 部署时跑这个 = hexo generate
  "outputDirectory": "public",       // 把 public/ 推到 CDN
  "installCommand": "npm install",
  "framework": null                  // Vercel 不要套用任何预设
}
```

`framework: null` 是关键——避免 Vercel 误认成 Next.js / 静态导出走它自己的逻辑。

### 5.5 `scaffolds/post.md`

`hexo new "标题"` 时会复制这个文件，把 `{{ title }}` 和 `{{ date }}` 替换。可以改默认 front matter（比如默认开 `katex: true`）。

### 5.6 `source/css/custom.css`

**这个项目的视觉灵魂**。重写了 Butterfly 默认配色（明润蓝 `#4584c7` + 暖金强调），加了多层阴影、字重、间距、悬停效果，做出"高级感"。

样式通过 `inject.head` 注入——所以会**覆盖**主题样式（CSS 加载顺序在主题后面）。

## 八、部署流程（Vercel 端发生了什么）

1. 你 `git push` 到 GitHub `main` 分支
2. GitHub 通过 webhook 通知 Vercel
3. Vercel 起一个临时 Node 容器
4. 跑 `npm install`（按 `package-lock.json` 装）
5. 跑 `npm run build`（即 `hexo generate`）
6. 把 `public/` 整个目录上传到 Vercel 的全球 CDN
7. 访客访问 `myblog-zeta-ruby-32.vercel.app` 时由最近的 CDN 节点返回静态文件

**整个流程约 30 秒到 1 分钟**。每次 push 都是全量重新构建（不是增量）。

如果只改了 `README.md` 这种不影响 `public/` 的文件，Vercel 仍然会构建一遍——开销很小。

## 十三、故障排查

**Q: 改了配置不生效？**
跑 `npm run clean && npm run server`，Hexo 有 `db.json` 缓存。

**Q: 本地正常线上 404？**
检查 `_config.yml` 里 `url` 是否正确。生成的绝对链接（RSS / sitemap / canonical）依赖这个值。

**Q: Vercel 构建失败？**
进 Vercel Dashboard → 对应 Deployment → Build Logs。最常见的是某个依赖没进 `package.json`（比如之前的 `hexo-generator-search`）。

**Q: YAML 报错？**
缩进必须用空格不能用 tab。冒号后面要有空格。字符串含特殊字符（`:`、`#`、`'`）时要加引号。

**Q: 改了 `_config.butterfly.yml` 没反应？**
确认是不是开了 `_config.butterfly.yml` 而不是 `node_modules/hexo-theme-butterfly/_config.yml`——后者每次 `npm install` 会被覆盖。前者优先级更高，是 Hexo 的"主题配置覆盖文件"。

**Q: 想完全改主题？**
`_config.yml` 改 `theme:` 字段，然后装对应的 npm 主题包。但 Butterfly 是 Hexo 生态里最强的主题之一，建议先用着。
